# Daily Learning Log

Create a folder for each day and store your work in it. Even if you didn't write any code that day, you should still create a Markdown (`.md`) file and document what you learned.

## Example

If your task was to learn **"How does JavaScript work under the hood?"**, your Markdown file could look like this:

```md
# How does JavaScript Work Under the Hood?

JavaScript is a single-threaded language that uses an event loop to handle asynchronous operations. I learned that it uses a call stack to execute functions, a heap to manage memory allocation, and a message queue to process asynchronous tasks. I also explored how the V8 engine compiles JavaScript into machine code, allowing it to run efficiently.
```

> **Note:** Write only what **you have learned** in your own words. Do **not** copy content from the internet. The goal is to reflect on your learning process and document your understanding.

If you wrote any JavaScript (`.js`) files while learning, include those files in the same daily folder. I attached an exmaple file as well.

## Purpose

The purpose of this exercise is to encourage self-reflection and reinforce your learning by documenting your understanding of the concepts you study.

By writing what you have learned in your own words, you:

- Strengthen your understanding of the topic.
- Build a personal knowledge base that you can revisit later.
- Identify areas where you need further clarification or practice.
- Develop the habit of documenting your learning consistently.
